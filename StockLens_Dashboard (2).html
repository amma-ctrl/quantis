Methodology

StockLens combines four fundamentally different forecasting approaches — deep learning, classical time-series, tree-based ensembles, and additive decomposition — then fuses their outputs through a weighted consensus engine to generate actionable buy/sell signals.

Forecasting Models

Deep Learning

LSTM Neural Network

A Long Short-Term Memory recurrent neural network trained on 5 years of daily OHLCV data plus 12 engineered technical features. The model learns temporal dependencies across a 60-day lookback window to produce 30-day forward price distributions. Weights are exported from Python/Keras and served via TensorFlow.js for fully client-side inference.

2 LSTM layers (128/64 units) Dropout 0.2 60-day lookback Adam optimizer RMSE 4.32 Confidence 78% Weight 0.30
Classical Time Series

ARIMA / SARIMA

A Seasonal AutoRegressive Integrated Moving Average model captures linear autocorrelation and deterministic seasonal cycles in the price series. Order parameters are selected via AIC minimization on a rolling training window. The seasonal component (period = 5 trading days / weekly) captures recurring Monday-effect and end-of-week patterns. Pre-computed in Python's statsmodels and served as static JSON.

SARIMA(1,1,1)(1,1,0,5) AIC 1842 Weekly seasonality RMSE 5.87 Confidence 62% Weight 0.20
Traditional Machine Learning

Random Forest / XGBoost

An ensemble of 200 decision trees trained on 28 feature-engineered technical indicators including momentum oscillators, moving average crossovers, and volume-weighted metrics. XGBoost gradient boosting refines residual errors from the base Random Forest. Features are importance-ranked at inference time; SMA_20 consistently dominates. Both models are pre-fitted in scikit-learn and served as JSON predictions.

200 trees (RF) XGBoost booster 28 features R² 0.87 RMSE 4.95 Weight 0.25
Additive Decomposition

Meta Prophet

Meta's Prophet library decomposes the price series into trend, yearly seasonality, weekly seasonality, and holiday effects using a piecewise linear trend with automatic changepoint detection. The additive structure makes forecasts interpretable — each component can be visualized and reasoned about independently. Strong Q4 seasonality is identified across US equity markets. Fitted in Python and serialized to JSON.

Yearly + weekly seasonality US holiday effects Auto changepoints MAPE 2.8% RMSE 5.10 Weight 0.25

Ensemble Consensus Engine

Each model casts a directional vote (BUY / HOLD / SELL) weighted by its historical validation accuracy. The consensus score determines the displayed signal and confidence bar. Weights are periodically recalibrated against rolling out-of-sample performance.

A BUY signal requires weighted consensus ≥ 0.60. A SELL signal requires weighted consensus ≥ 0.60 in the opposite direction. Anything between is classified as HOLD. Trade entry/exit zones are derived by averaging model price-target distributions and intersecting with support/resistance levels from the price history.

Backtesting Approach

Each model is evaluated over a 12-month rolling out-of-sample window with $10,000 starting capital, no leverage, and realistic 0.1% slippage per trade. The buy-and-hold benchmark uses a single entry at window open with no rebalancing.
